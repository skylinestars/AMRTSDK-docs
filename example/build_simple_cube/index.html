示例 07: 简单立方体构建¶

功能说明¶

本示例展示如何使用 AmrtBuilder API 从代码中构建一个带多个贴图的立方体模型，并保存为AMRT格式。

主要演示内容¶

  1. 创建网格数据：手动定义立方体的顶点、面、法线和UV坐标
  2. 多材质支持：为立方体的6个面分别设置不同的材质和贴图
  3. 独立网格架构：每个面使用独立的网格，避免前端切线计算问题
  4. 使用Builder API：展示AmrtBuilder的基本使用流程
  5. 配置构建选项：设置压缩、输出格式等选项
  6. 保存AMRT文件：将构建的场景保存为分布式格式
  7. 贴图文件处理：自动复制贴图文件到输出目录

编译和运行¶

编译¶

cd build
cmake ..
cmake --build . --target build_simple_cube

运行¶

# Windows
bin\examples\build_simple_cube.exe [输出路径]

# Linux/macOS
./bin/examples/build_simple_cube [输出路径]

示例¶

# 默认输出到 output_cube 目录
build_simple_cube

# 指定输出路径
build_simple_cube my_cube

输出结果¶

立方体面的配置¶

每个面有独立的网格、材质和贴图：

面 名称 法线方向 颜色 贴图
后面 Red_Back_Z- (0,0,-1) 红色 1.png
前面 Green_Front_Z+ (0,0,1) 绿色 2.png
左面 Blue_Left_X- (-1,0,0) 蓝色 3.png
右面 Yellow_Right_X+ (1,0,0) 黄色 4.png

代码要点¶

1. 创建单个面的网格¶

MeshData CreateFaceMesh(int faceIndex, const std::string& name, 
                       const Vec4& color, const Vec3& normal, 
                       const std::vector<Vec3>& faceVertices)
{
    MeshData mesh;
    mesh.vertices = faceVertices;  // 4个顶点
    mesh.indices = {0, 1, 2, 0, 2, 3};  // 2个三角形
    mesh.normals = {normal, normal, normal, normal};
    mesh.uvs0 = {{0,0}, {1,0}, {1,1}, {0,1}};  // 标准UV映射
    return mesh;
}

2. 为每个面创建材质¶

MaterialData material;
material.name = faceName;
material.baseColor = Vec4(1, 1, 1, 1);  // 白色底色让贴图正常显示
material.diffuseTexture.hasTexture = true;
material.diffuseTexture.path = "cubeImage/1.png";
builder->AddMaterial(material);

3. 构建场景树¶

// 创建根节点
auto rootNodeId = builder->AddNode("CubeRoot", -1, Matrix4x4::Identity());

// 为每个面创建子节点和网格
for (每个面) {
    auto materialId = builder->AddMaterial(material);
    auto meshId = builder->AddMesh(faceMesh);
    auto nodeId = builder->AddNode(faceName, rootNodeId, Matrix4x4::Identity());
    builder->AttachMeshToNode(nodeId, meshId);
}

4. 获取统计信息¶

BuildStatistics stats = builder->GetStatistics();
std::cout << "节点数: " << stats.nodesAdded << std::endl;    // 7 (1个根 + 6个面)
std::cout << "网格数: " << stats.meshesAdded << std::endl;   // 6
std::cout << "顶点数: " << stats.totalVertices << std::endl; // 24
std::cout << "三角形数: " << stats.totalTriangles << std::endl; // 12

重要说明¶

为什么使用独立网格而不是SubMesh？¶

本示例采用每个面一个网格的架构，而不是一个网格多个SubMesh，原因是：

  1. 前端兼容性：某些前端加载器在处理多SubMesh时，切线（Tangent）计算会失败
  2. 灵活性：每个面可以独立控制（显示/隐藏、材质替换等）
  3. 调试便利：问题定位更容易

对于简单模型（如立方体），这种架构的性能影响可以忽略不计。

测试文件¶

  • main.cpp：完整的立方体示例（6个面，6个材质）
  • main_single_face.cpp：简化版测试（1个面，1个材质）

可以先运行 main_single_face.cpp 验证基本功能，再运行完整版本。

故障排除¶

1. 贴图文件未找到¶

错误信息： 

⚠ 警告: 找不到贴图源目录

解决方案： - 确保 cubeImage 目录与可执行文件在同一目录 - 或在源码目录运行（AmrtSdk/examples/build_simple_cube/） - 或手动将贴图复制到输出目录

2. 前端加载时切线计算错误¶

错误信息： 

Uncaught TypeError: Cannot read properties of undefined (reading 'add')
    at handleTriangle
    at Object.computeTangents

原因：使用了多SubMesh结构（旧版本）

解决方案：使用当前版本（独立网格架构）

3. 包围盒过大导致渲染错误¶

解决方案：立方体尺寸设置为 [-0.5, 0.5]，避免过大的包围盒
