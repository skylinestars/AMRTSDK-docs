示例13：导出带动画的AMRT文件为FBX格式¶

功能说明¶

本示例展示如何： 1. 使用 AmrtReader 读取包含动画的 AMRT 文件 2. 打印模型信息（网格、材质、节点） 3. 打印动画信息（骨骼结构、动画通道） 4. 将骨骼、蒙皮和动画数据导出为 FBX 格式

使用方法¶

编译¶

cd build
cmake --build . --config Debug --target export_anim_to_fbx

或者使用 Release 配置：

cmake --build . --config Release --target export_anim_to_fbx

运行¶

# 在 Windows 上（Debug）
.\bin\Debug\export_anim_to_fbx.exe <输入AMRT路径> <输出FBX文件>

# 示例
.\bin\Debug\export_anim_to_fbx.exe D:\Data\model\amrt\character D:\Data\model\fbx\character.fbx
# 在 Windows 上（Release）
.\bin\Release\export_anim_to_fbx.exe <输入AMRT路径> <输出FBX文件>

输出信息¶

程序会输出以下信息：

1. 模型信息¶

  • 节点数量
  • 网格信息（名称、顶点数、三角形数、子网格数）
  • 材质信息（名称、贴图路径）

2. 动画信息¶

  • 骨骼数量和层级结构
  • 动画数量和详细信息：
  • 动画名称
  • 时长（秒）
  • 帧率（FPS）
  • 位移/旋转/缩放通道数
  • 骨骼通道数

3. 转换过程¶

  • FBX SDK 初始化
  • 材质转换
  • 骨骼结构创建
  • 网格和蒙皮数据转换
  • 动画数据创建
  • FBX 文件导出

支持的功能¶

模型数据¶

  • ✅ 顶点位置
  • ✅ 顶点法线
  • ✅ UV 坐标
  • ✅ 顶点颜色
  • ✅ 材质和贴图

骨骼系统¶

  • ✅ 骨骼层级结构
  • ✅ 骨骼变换矩阵
  • ✅ 蒙皮权重数据
  • ✅ 顶点-骨骼绑定

动画系统¶

  • ✅ 位移动画（Translation）
  • ✅ 旋转动画（Rotation，四元数转欧拉角）
  • ✅ 缩放动画（Scale）
  • ✅ 骨骼动画
  • ✅ 关键帧时间和值

依赖要求¶

  • FBX SDK: 必须安装 Autodesk FBX SDK 2020.3.7 或更高版本
  • AMRT SDK: 需要编译好的 AMRT SDK 库
  • C++17: 需要支持 C++17 标准的编译器

技术细节¶

骨骼转换¶

  • AMRT 的 BoneData → FBX 的 FbxSkeleton 和 FbxNode
  • 根骨骼设置为 FbxSkeleton::eRoot
  • 子骨骼设置为 FbxSkeleton::eLimbNode
  • 使用 bindPoseMatrix 设置骨骼的绑定姿态

蒙皮转换¶

  • AMRT 的 SkinData → FBX 的 FbxSkin 和 FbxCluster
  • 为每个影响网格的骨骼创建一个 FbxCluster
  • 设置顶点权重到对应的 FbxCluster
  • 使用 eTotalOne 链接模式（权重总和为1）

动画转换¶

  • AMRT 的 AnimationData → FBX 的 FbxAnimStack 和 FbxAnimLayer
  • 位移通道 → LclTranslation 曲线
  • 旋转通道 → LclRotation 曲线（四元数自动转换为欧拉角）
  • 缩放通道 → LclScaling 曲线
  • 设置动画时间范围（起始时间 + 持续时间）

注意事项¶

  1. 输入文件格式: 必须是有效的 AMRT 分布式格式（包含 fileList.json）
  2. 骨骼数据: 如果 AMRT 文件不包含骨骼数据，将只导出静态模型
  3. 动画数据: 如果 AMRT 文件不包含动画数据，将只导出绑定姿态
  4. 贴图路径: 相对路径会自动拼接 AMRT 文件的目录路径
  5. 四元数转换: 旋转动画会自动将四元数转换为欧拉角（FBX 标准）

  • AmrtSdk/src/core/io/skeleton_loader.h - 骨骼加载器

故障排除¶

问题：找不到骨骼数据¶

解决方案: - 确认 AMRT 文件包含骨骼数据（查看 fileList.json 中的 bone 字段） - 确认骨骼数据文件存在且可读

问题：动画导出为空¶

解决方案: - 确认动画通道的 targetNodeId 与场景节点ID匹配 - 检查关键帧数据是否有效 - 确认动画时长不为0

问题：蒙皮变形不正确¶

解决方案: - 确认每个顶点的权重总和为1.0 - 检查骨骼ID是否与骨骼数据匹配 - 确认骨骼变换矩阵正确
