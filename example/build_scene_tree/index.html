示例08：场景树构建¶

功能说明¶

本示例演示如何使用 AMRT Builder API 构建层级化的场景树结构。通过创建一个简化的太阳系模型（太阳-地球-月球）来展示：

  1. 场景树层级关系：父子节点结构
  2. 变换矩阵继承：子节点的变换相对于父节点
  3. 材质系统：为每个网格配置不同的材质
  4. 完整的数据输出：生成网格文件、材质文件和文件列表

场景结构¶

太阳 (原点 0,0,0)
└── 地球 (相对太阳: +10,0,0 → 世界坐标: 10,0,0)
    └── 月球 (相对地球: +3,0,0 → 世界坐标: 13,0,0)

关键修复内容¶

1. API 错误修复¶

  • ✅ MeshData.hasNormals → MeshData.flags = MESH_FLAG_HAS_NORMALS
  • ✅ Matrix4x4.m[12] → Matrix4x4.m[0][3] (二维数组访问)
  • ✅ AmrtBuilder builder → auto builder = AmrtBuilder::Create(options)
  • ✅ options.outputFormat → options.format
  • ✅ AddNode(NodeData) → AddNode(name, parentId, transform)
  • ✅ stats.nodeCount → stats.nodesAdded

2. 材质系统集成¶

问题：之前的版本没有添加任何材质，导致只生成了 mesh_0.mrmb，没有生成 mtl.json。

修复： 1. 为每个天体创建独立的材质（太阳黄色、地球蓝色、月球灰色） 2. 调用 builder->AddMaterial() 添加材质到构建器 3. 为每个网格设置 SubMeshMaterial，关联材质ID

// 创建材质
MaterialData sunMaterial;
sunMaterial.name = "Sun_Material";
sunMaterial.baseColor = Vec4(1.0f, 1.0f, 0.0f, 1.0f); // 黄色
sunMaterial.hasEmissive = true; // 自发光效果
auto matId = builder->AddMaterial(sunMaterial);

// 关联材质到网格
MeshData sunMesh = CreateSphereMesh(...);
SubMeshMaterial subMesh;
subMesh.materialIndex = matId.GetValue();
subMesh.startFaceIndex = 0;
subMesh.endFaceIndex = 8; // 三角形数量
sunMesh.subMeshes.push_back(subMesh);

输出文件¶

运行程序后会在指定目录生成以下文件：

output_solar_system/
├── fileList.json      # 文件列表（场景描述）
├── mtl.json           # 材质数据（3个材质）
├── mesh_0.mrmb        # 太阳网格
├── mesh_1.mrmb        # 地球网格
└── mesh_2.mrmb        # 月球网格

构建和运行¶

使用 CMake（推荐）¶

# 进入构建目录
cd build2022Debug

# 构建项目
cmake --build . --config Debug --target build_scene_tree

# 运行示例
.\bin\examples\build_scene_tree.exe

# 或指定输出路径
.\bin\examples\build_scene_tree.exe my_solar_system

查看输出¶

构建完成后，输出目录结构：

output_solar_system/
  fileList.json - 文件列表和场景配置
  mtl.json      - 3个材质的完整定义
  mesh_0.mrmb   - 太阳球体（黄色自发光）
  mesh_1.mrmb   - 地球球体（蓝色）
  mesh_2.mrmb   - 月球球体（灰色）

核心概念¶

1. 场景树（Scene Tree）¶

场景树是一个层级结构，子节点的变换相对于父节点： - 地球距离太阳 10 个单位（X轴） - 月球距离地球 3 个单位（X轴） - 月球的世界坐标是 (13, 0, 0)，因为变换沿树累积

2. 变换矩阵（Transform Matrix）¶

Matrix4x4 CreateTranslationMatrix(float x, float y, float z)
{
    Matrix4x4 mat = Matrix4x4::Identity();
    mat.m[0][3] = x;  // 第4列第1行
    mat.m[1][3] = y;  // 第4列第2行
    mat.m[2][3] = z;  // 第4列第3行
    return mat;
}

AMRT 使用行主序矩阵，平移分量在第4列。

3. 材质系统（Material System）¶

每个材质包含： - 基础颜色：baseColor / diffuseColor - PBR 属性：roughness (粗糙度), metallic (金属度) - 自发光：hasEmissive + emissiveColor - 纹理贴图：可选的各种贴图（漫反射、法线等）

4. 网格-材质关联¶

通过 SubMeshMaterial 将材质关联到网格的子网格： - materialIndex：材质ID（来自 AddMaterial 返回值） - startFaceIndex：起始三角形索引 - endFaceIndex：结束三角形索引

技术要点¶

Builder 使用流程¶

// 1. 创建 Builder
auto builder = AmrtBuilder::Create(options);

// 2. 开始构建
builder->Begin();

// 3. 添加材质
auto matId = builder->AddMaterial(material);

// 4. 添加网格（关联材质）
mesh.subMeshes.push_back(subMesh);
auto meshId = builder->AddMesh(mesh);

// 5. 添加节点（构建场景树）
auto parentId = builder->AddNode("Parent", -1, transform);
auto childId = builder->AddNode("Child", parentId, childTransform);

// 6. 附加网格到节点
builder->AttachMeshToNode(nodeId, meshId);

// 7. 保存
builder->Save(outputPath);

关键 API¶

API 说明
AmrtBuilder::Create(options) 创建 Builder 实例
builder->Begin() 开始构建会话
builder->AddMaterial(material) 添加材质，返回材质ID
builder->AddMesh(mesh) 添加网格，返回网格ID
builder->AddNode(name, parentId, transform) 添加节点，返回节点ID
builder->AttachMeshToNode(nodeId, meshId) 关联网格到节点
builder->Save(path) 保存到文件/目录
builder->GetStatistics() 获取构建统计信息

常见问题¶

Q: 为什么需要 SubMeshMaterial？¶

A: SubMeshMaterial 是网格和材质之间的桥梁，它告诉渲染器： - 哪些三角形使用哪个材质 - 支持一个网格使用多个材质（如立方体6个面6个材质）

Q: 为什么没有生成 mtl.json？¶

A: 因为没有调用 AddMaterial() 添加材质。只有添加了材质，Builder 才会生成材质文件。

Q: parentId 为 -1 是什么意思？¶

A: -1 表示该节点是根节点，没有父节点。它会被直接挂在场景根下。

Q: 变换矩阵如何累积？¶

A: 子节点的世界变换 = 父节点的世界变换 × 子节点的局部变换。 例如：月球世界位置 = 太阳变换 × 地球变换 × 月球变换 = (0) + (10) + (3) = 13
