示例 10: 复杂场景构建¶

功能说明¶

本示例展示如何使用 AmrtBuilder 构建一个完整的复杂场景，包含多个物体、层级节点和材质。这是一个综合性示例，整合了前面所有示例的知识点。

场景描述¶

一个简单的房间场景，包含： - 地板 - 20m x 20m的木地板 - 桌子 - 包含桌面和4条桌腿，使用深色木材 - 2把椅子 - 每把椅子包含座位和4条椅腿，使用浅色木材

主要演示内容¶

  1. 多物体场景：创建多个不同类型的物体
  2. 层级结构：使用父子节点组织复杂物体（桌子、椅子）
  3. 网格复用：相同的腿部网格被多次使用
  4. 材质管理：为不同物体应用不同材质
  5. 变换组合：位置、旋转的复杂变换
  6. 场景组织：清晰的命名和层级结构

场景树结构¶

Room (根节点)
├── Floor (地板)
├── Table (桌子组)
│   ├── TableTop (桌面)
│   ├── TableLeg1 (桌腿1)
│   ├── TableLeg2 (桌腿2)
│   ├── TableLeg3 (桌腿3)
│   └── TableLeg4 (桌腿4)
├── Chair1 (椅子1组)
│   ├── Seat (座位)
│   ├── ChairLeg1 (椅腿1)
│   ├── ChairLeg2 (椅腿2)
│   ├── ChairLeg3 (椅腿3)
│   └── ChairLeg4 (椅腿4)
└── Chair2 (椅子2组)
    ├── Seat (座位)
    ├── ChairLeg1 (椅腿1)
    ├── ChairLeg2 (椅腿2)
    ├── ChairLeg3 (椅腿3)
    └── ChairLeg4 (椅腿4)

编译和运行¶

编译¶

cd build
cmake ..
cmake --build . --target build_complex_scene

运行¶

# Windows
bin\examples\build_complex_scene.exe [输出路径]

# Linux/macOS
./bin/examples/build_complex_scene [输出路径]

示例¶

# 默认输出到 output_complex_scene 目录
build_complex_scene

# 指定输出路径
build_complex_scene my_room

输出结果¶

程序会在指定目录下生成： - fileList.json - 文件列表 - mtl.json - 4种材质定义 - mesh_1.mrmb 至 mesh_6.mrmb - 6种网格数据

代码要点¶

1. 网格复用策略¶

// 创建桌腿网格（只创建一次）
MeshData tableLegMesh = CreateBoxMesh(meshIdCounter++, "TableLeg", 0.2f, 1.5f, 0.2f);
builder.AddMesh(tableLegMesh);

// 在4个不同的节点复用同一个网格
for (size_t i = 0; i < 4; ++i) {
    NodeData legNode;
    // ... 设置不同的位置
    builder.AttachMeshToNode(legNode.nodeId, tableLegMesh.meshId);
}

2. 层级节点组织¶

// 创建父节点（桌子组）
NodeData tableRootNode;
tableRootNode.nodeId = nodeIdCounter++;
tableRootNode.name = "Table";
builder.AddNode(tableRootNode);

// 创建子节点（桌面）
NodeData tableTopNode;
tableTopNode.parentId = tableRootNode.nodeId;  // 指定父节点
builder.AddNode(tableTopNode);
builder.SetNodeParent(tableTopNode.nodeId, tableRootNode.nodeId);

3. 材质批量应用¶

// 为所有桌子部件应用相同材质
builder.SetMeshMaterial(tableTopMesh.meshId, tableMat.materialId);
builder.SetMeshMaterial(tableLegMesh.meshId, tableMat.materialId);

性能优化要点¶

  1. 网格复用：相同的腿部网格被复用12次（4个桌腿 + 8个椅腿）
  2. 材质共享：相同材质的物体共享材质定义
  3. 层级变换：使用父子节点避免重复计算变换
  4. 批量处理：统一创建网格和材质，再构建场景

实际应用场景¶

本示例展示的模式适用于： - 建筑可视化：房间、家具、装饰品 - 游戏场景：关卡、道具、环境 - 产品展示：产品组件、装配体 - 工业设计：机械装配、零件组合

最佳实践¶

  1. 命名规范：使用清晰有意义的名称
  2. 层级组织：逻辑相关的物体归为一组
  3. 网格复用：相同几何体只创建一次
  4. 材质管理：预先创建所有材质
  5. ID分配：使用计数器避免ID冲突
  6. 错误处理：每步操作都检查返回值

扩展建议¶

基于本示例，你可以： 1. 添加更多家具类型 2. 添加光源和阴影 3. 添加墙壁和天花板 4. 实现房间布局生成器 5. 添加纹理和细节材质 6. 实现动画（门开关、抽屉等）
