示例11：合并分布式AMRT文件为单个文件¶

功能说明¶

本示例演示如何使用AmrtMerger将分布式AMRT文件（包含fileList.json和多个数据文件）合并为单个.amrt文件。

使用场景¶

  • 优化网络传输（单个文件比多个小文件更高效）
  • 简化文件管理
  • 支持内存映射加载
  • 便于分发和部署

编译¶

cd build
cmake --build . --target example_11_merge_to_one_file

运行¶

基本用法¶

./example_11_merge_to_one_file <输入AMRT文件夹> <输出文件路径>

示例¶

# 不压缩合并
./example_11_merge_to_one_file D:/models/my_model/ D:/output/model.amrt

# 压缩合并
./example_11_merge_to_one_file D:/models/my_model/ D:/output/model.amrt 1

代码示例¶

方法1：使用便捷函数（推荐）¶

#include <amrt/amrt.h>

std::string errorMsg;
if (amrt::MergeAmrtToOneFile(
    "input/folder/",      // 输入AMRT文件夹
    "output/model.amrt",  // 输出文件
    false,                // 是否压缩
    &errorMsg))           // 错误信息
{
    std::cout << "合并成功！" << std::endl;
}
else
{
    std::cerr << "合并失败: " << errorMsg << std::endl;
}

方法2：使用AmrtMerger类（支持进度监控）¶

#include <amrt/amrt.h>

amrt::AmrtMerger merger;

// 设置进度回调
merger.SetProgressCallback([](size_t current, size_t total, const std::string& msg) {
    std::cout << "[" << current << "/" << total << "] " << msg << std::endl;
    return true; // 返回false可取消操作
});

// 加载AMRT文件夹
if (!merger.Load("input/folder/", false))
{
    std::cerr << "加载失败: " << merger.GetLastError() << std::endl;
    return;
}

// 分析所有文件
if (!merger.AnalyzeAllFiles())
{
    std::cerr << "分析失败: " << merger.GetLastError() << std::endl;
    return;
}

// 获取统计信息
size_t totalFiles;
uint64_t totalBytes;
merger.GetStatistics(totalFiles, totalBytes);
std::cout << "总文件数: " << totalFiles << std::endl;
std::cout << "总大小: " << (totalBytes / 1024.0 / 1024.0) << " MB" << std::endl;

// 保存为单个文件
if (!merger.SaveToOneFile("output/model.amrt"))
{
    std::cerr << "保存失败: " << merger.GetLastError() << std::endl;
    return;
}

std::cout << "合并成功！" << std::endl;

API说明¶

AmrtMerger类¶

方法 说明
Load(folder, useCompress) 加载分布式AMRT文件夹
AnalyzeAllFiles() 分析所有文件并准备合并
SaveToOneFile(output) 保存为单个AMRT文件
SetProgressCallback(callback) 设置进度回调函数
GetStatistics(files, bytes) 获取文件统计信息
GetBoundingBox() 获取场景包围盒
GetLastError() 获取最后的错误信息
IsLoaded() 是否已加载
IsAnalyzed() 是否已分析
Reset() 重置状态

便捷函数¶

bool MergeAmrtToOneFile(
    const std::string& amrtFolderPath,  // 输入文件夹
    const std::string& outputFilePath,  // 输出文件
    bool useCompression = false,        // 是否压缩
    std::string* errorMsg = nullptr     // 错误信息（可选）
);

文件格式¶

单文件AMRT格式结构¶

[文件头]
  - Magic: "amrt" (4字节)
  - Version: 1.0 (4字节 float)
  - Flags: (4字节 uint32)
  - HeaderSize: (4字节 uint32)

[文件索引]
  - 文件类型1 (字符串)
    - 文件数量 (4字节)
    - 文件1信息 (名称、偏移、大小)
    - 文件2信息...
  - 文件类型2...

[包围盒]
  - "box" (字符串)
  - Min点 (3个float)
  - Max点 (3个float)

[对齐填充]

[文件数据块1]
[对齐填充]
[文件数据块2]
...

注意事项¶

  1. 输入要求：输入文件夹必须包含有效的fileList.json文件
  2. 内存使用：合并过程会将所有文件内容加载到内存，请确保有足够的内存
  3. 压缩选项：启用压缩需要LZ4库支持
  4. 文件对齐：所有数据块都会进行4字节对齐
  5. 错误处理：建议检查每个步骤的返回值并处理错误

性能提示¶

  • 对于大文件，建议使用进度回调监控进度
  • 压缩可以减小文件大小，但会增加处理时间
  • 单文件格式更适合网络传输和内存映射
  • 分布式格式更适合开发和调试
